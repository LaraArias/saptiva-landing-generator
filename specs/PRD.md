# Product Requirements Document - Saptiva Landings

## Objetivo
Generar landing pages que conviertan visitantes en leads calificados para cada vertical de Saptiva.

## Reglas Globales

### Principios de Diseño
1. **Mobile-first**: Diseñar primero para móvil, luego adaptar a desktop
2. **Performance**: Carga < 3 segundos
3. **Accesibilidad**: WCAG 2.1 AA mínimo
4. **Consistencia**: Usar brand assets definidos

### Principios de Contenido
1. **Outcome-driven**: Enfocarse en resultados, no features
2. **Segmento-específico**: Adaptar mensaje al driver del dominio
3. **No inventar**: Todo copy viene del contexto
4. **CTA claro**: Un solo call-to-action principal por página

## Estructura Base

Todas las landings deben incluir:

### Obligatorio
1. **Hero**: Propuesta de valor + CTA principal
2. **Problema**: Pain points del segmento
3. **Solución**: Cómo Saptiva resuelve
4. **Beneficios**: 3-4 beneficios clave
5. **CTA Final**: Refuerzo del call-to-action

### Opcional (según vertical)
- Testimonios / Logos de clientes
- Casos de éxito
- Comparativa / Diferenciadores
- FAQ
- Recursos descargables

## Restricciones

### No incluir
- Precios (requiere conversación)
- Features técnicos detallados
- Comparativas directas con competidores
- Promesas no respaldadas por contexto

### Requiere aprobación
- Testimonios de clientes (verificar permiso)
- Logos de empresas (verificar permiso)
- Métricas específicas (verificar fuente)
